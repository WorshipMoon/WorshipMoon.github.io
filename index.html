本地hexo同步案例与和github

环境需要

  1. node

  2. git

    • 全局安装 npm install -g hexo-cli

    • 局部安装npm install hexo
      安装以后，可以使用以下两种方式执行 Hexo：

      1. npx hexo <command>
      2. 将 Hexo 所在的目录下的 node_modules 添加到环境变量之中即可直接使用 hexo <command>：

  4. 安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

好，现在爬坑开始

我想的是阿里云和github都能访问，github不让百度抓，我也有服务器和备案号的域名，两个都能访问就可以了，现在目的就是一篇博文两边同步

先记住两个命令

hexo clean && hexo g && hexo s

这条命令 清理缓存 生成静态 启动服务 &&连接符可以换成;本地执行完这个服务就跑起来了，当然生成静态public文件夹之后，用域名指向这个这个目录访问完全没问题，也可以启动服务之后，反向代理到这个端口访问也没问题，这个需要添加守护进程

hexo clean;hexo generate;hexo deploy

这条命令 清理缓存 生成静态 按照配置部署，配置里面也上github部署地址，这个后面讲到
